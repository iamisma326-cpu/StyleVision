# Plan: Optimize assets for faster Vercel deployment

## Problem
- Images: 368KB-2.5MB each, resolutions up to 6000x7360 px
- Videos: 23-49MB each (~120MB total)
- Total: ~130MB of assets causing slow loading

## Solution

### 1. Optimize images
- Resize to max 1200px width (maintain aspect ratio)
- Convert to WebP format (~50-70% smaller than JPEG)
- Save to `assets/img/` replacing originals

```bash
for img in assets/img/*.jpg; do
  magick "$img" -resize "1200x>" -quality 80 "${img%.jpg}.webp"
done
```

### 2. Compress videos
- Use ffmpeg to reduce video size
- Target: ~2-5MB per video

```bash
for vid in assets/img/*.mp4; do
  ffmpeg -i "$vid" -vcodec h264 -crf 28 -preset slow "$vid.tmp.mp4" && mv "$vid.tmp.mp4" "$vid"
done
```

### 3. Update HTML files
- Change image references from `.jpg` to `.webp`
- Files to update: `html/index.html`, `html/como-funciona.html`, `html/contacto.html`, `html/explorar.html`, `html/herramienta-ia.html`

### 4. Add lazy loading
- Add `loading="lazy"` to images below the fold
- Keep hero image eager-loaded

### 5. Commit and push

## Expected results
- Images: 50-70% smaller (1.5MB → ~300-500KB)
- Videos: 60-80% smaller (49MB → ~5-10MB)
- Total: ~130MB → ~20-30MB

## Verification
- Check file sizes after optimization
- Deploy to Vercel and verify loading speed
