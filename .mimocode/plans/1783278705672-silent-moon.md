# Plan: Fix Vercel static asset serving

## Root Cause
Vercel rewrites are catching all routes including static files. The rewrites in vercel.json match `/` and other routes, but static files at `/assets/`, `/css/`, `/js/` should be served directly.

## Solution
Restructure project to use Vercel's `public/` directory convention:
- Vercel automatically serves files from `public/` at the root level
- Keep HTML in `html/`
- Move css/, js/, assets/ inside public/

## Steps
1. Create `public/` directory
2. Move `css/`, `js/`, `assets/` inside `public/`
3. Update vercel.json with rewrites
4. Update all HTML paths to remove leading `/` (since public/ serves at root)
5. Commit and push

## Files to modify
- vercel.json
- html/index.html
- html/como-funciona.html
- html/explorar.html
- html/contacto.html
- html/herramienta-ia.html
